File system

Each folder with index.html is a valid URL to connect to from root. In each folder, it is possible to provide with custom style.css to overwrite the default /style.css.

When creating a new file that you wish to show up in the Release, you must explcitly add it to build/build.js

When creating a new file, it should be named with singular noun with spaces replaced with hypens. It is script.js, not scripts.js. Folders and URL folders are not subject to this convention.

Translations

Translation requires you to build the website with the build script.

In each folder, there will be an index.html files to dictate the layout and the translation keys. In each folder, there will be .json files with language code.

Translation keys will be replaced with translated content in that perspective language, then put into the correct folder according to the key URL. Translation key should be in English.

Because the replace is pure plain text, it is recommened to put translation key inside >translationKey< for keys with confusable name. The best practice is to name keys with long enough snake_case name. Snake case is to prevent conflict with CSS camel case.

A certain control key will be ignored (like URL)

Build script

The build script is important in a sense that it will put all html-src into the element itself, removing the need to execute script to fetch, hence help with SEO because the meta tags are present.

The build script is Static Wind which can be found here on GitHub.

Build config in /Static-Wind/config.json

{ "languages": ["en", "vi"], "masterTranslation": "comp/", "releaseItems": [ "asset", "about", "about/importance", "about/programs", "dev", "home", "resources", "404.html", "index.html", ".GITIGNORE", "CNAME", "LICENSE", "robots.txt", "sitemap.txt" ] }

Development and Deployment

Please carefully check Static Wind's README to grasp the understand of the build process. In short, the build process will resolve all translation string and html-src before copy it to the Release branch.

To publish code, run node Static-Wind/build.js with the flag R. (You have to manually type it out for security reasons)
